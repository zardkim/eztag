작성일: 2026-03-25

## 1. 목표

현재 단순 테이블 형태의 HTML 출력을 벅스뮤직·멜론 앨범 상세 페이지 수준의 UI로 개선한다.
표시 항목은 현행과 동일하게 유지하되, 레이아웃·시각 디자인·모바일 반응형을 전면 재설계한다.

## 2. 현재 구조 분석

### 2-1. 현재 표시 항목 (유지 필수)

| 영역 | 항목 |
|------|------|
| 앨범 헤더 | 커버아트, 앨범 타입(Album), 앨범 제목, 앨범 아티스트, 연도, 장르, 레이블, 트랙 수, 총 재생시간 |
| 앨범 소개 | description (있을 때만) |
| 트랙 테이블 | 디스크, 트랙번호, 제목, 타이틀 뱃지, 가사 여부, 아티스트, 앨범아티스트*, 앨범명*, 연도*, 장르*, 발매일*, ISRC*, 설명(comment)*, 재생시간, MV(YouTube)*, 비트레이트*, 샘플레이트*, 파일크기* |
| 푸터 | eztag 배지, 생성 시간, 소스 폴더명 |

> `*` 해당 데이터 존재 시에만 컬럼 표시 (현행 유지)

### 2-2. 현재 문제점

- 앨범 헤더가 평범한 카드 수준으로 음악 서비스다운 임팩트가 없음
- 트랙 테이블이 좌우 스크롤은 되지만 모바일에서 가독성 저하
- PC/모바일 레이아웃 차이가 미미함
- 컬러 스킴이 단조로움 (흰/회색 계열)
- 아티스트 강조, 재생 버튼 영역 등 음악 서비스 특유의 요소 부재

## 3. 참고 디자인 — Bugs/Melon 주요 특징

### Bugs Music 앨범 상세
- 상단 전체폭 배너: 커버아트 블러 배경 + 좌측 커버 + 우측 텍스트 정보
- 커버아트: 큰 사이즈(PC 240px, 모바일 160px), 그림자 강조
- 앨범 정보: 타입 뱃지(정규/EP/싱글) > 앨범명(대제목) > 아티스트(컬러 강조) > 발매일·장르·레이블
- 트랙 목록: 번호 | 재생버튼 | 제목+타이틀뱃지 | 아티스트 | 시간 (행 hover 시 배경색 변화)
- 타이틀곡 특별 강조 (배경색 또는 아이콘 차별화)

### Melon 앨범 상세
- 커버 좌측 고정 + 우측 상세 정보 (PC)
- 모바일: 커버 중앙 정렬, 정보 세로 스택
- 트랙 목록: 번호 좌측, 제목 중앙, 시간 우측 — 3열 구조 (모바일 최적화)
- 가사 있는 곡 아이콘, 타이틀곡 강조색
- 섹션 구분선 + 섹션 제목(트랙 목록, 앨범 소개)

## 5. 세부 구성 요소 설계

### 5-1. Hero Banner (앨범 헤더 교체)

```html
<div class="hero">
  <div class="hero-bg" style="background-image: url(커버)"></div>  <!-- 블러+어둠 오버레이 -->
  <div class="hero-inner">
    <div class="hero-cover"><img ...></div>
    <div class="hero-info">
      <span class="album-type-badge">정규앨범</span>
      <h1 class="hero-title">앨범명</h1>
      <a class="hero-artist">아티스트명</a>
      <div class="hero-meta">2024 · 팝 · 레이블명</div>
      <div class="hero-stats">12곡 · 총 48:32</div>
    </div>
  </div>
</div>
```

**CSS 핵심:**
- `hero-bg`: `position: absolute; inset: 0; background-size: cover; filter: blur(40px) brightness(0.4) saturate(1.2)`
- `hero-inner`: `position: relative; z-index: 1; max-width: 1200px; margin: auto; padding: 40px 24px; display: flex; gap: 32px; align-items: flex-end`
- `hero-cover`: `width: 240px; border-radius: 16px; box-shadow: 0 20px 60px rgba(0,0,0,0.5)`
- `hero-title`: `font-size: 32px; font-weight: 900; color: #fff; text-shadow: 0 2px 8px rgba(0,0,0,0.5)`
- `hero-artist`: `font-size: 18px; color: rgba(255,255,255,0.85); font-weight: 600`
- 모바일: `flex-direction: column; align-items: center; text-align: center`

### 5-2. 앨범 타입 뱃지

커버 이미지 없을 때와 있을 때 모두 표시.
`"Album"` 하드코딩 대신 트랙 수 기반 자동 분류:
- 1곡: 싱글 (Single)
- 2~5곡: EP
- 6곡+: 정규 (Album)

### 5-3. 트랙 목록 — PC 테이블

현행 컬럼 구조 유지하되 시각적 개선:

| 변경 전 | 변경 후 |
|---------|---------|
| 단색 헤더 | 투명 헤더 + 하단 구분선만 |
| 제목 단일 행 | 제목(굵음) + 아래 아티스트(작음, 흐림) — 2행 구조 |
| 트랙번호 `01` 숫자 | hover 시 숫자 → ▶ 아이콘 교체 (CSS only) |
| 타이틀뱃지 주황 그라디언트 | 멜론식 `T` 뱃지 (빨간 원형) |
| 가사 보라 점 | 가사 아이콘 `♪` 또는 `L` 뱃지 (작은 파란 pill) |
| YouTube 열 분리 | YouTube 열 유지, 아이콘 개선 |

### 5-4. 트랙 목록 — 모바일 카드형

모바일에서는 테이블 대신 카드 리스트로 전환:

```
[번호] [제목 + 타이틀뱃지 + 가사] [시간]
       [아티스트]
```

- 기술 정보 (비트레이트·샘플레이트·파일크기·ISRC·발매일·설명 등): `<details>` 아코디언으로 각 트랙 하단에 수납
- 테이블은 `display: none` on mobile, 카드 리스트는 `display: none` on desktop
- 이렇게 하면 데이터 손실 없이 모바일 가독성 확보

### 5-5. 앨범 소개 섹션

- PC: 현행 `desc-box` 유지 (좌우 여백 카드)
- 모바일: `<details>` 접기/펼치기로 변경하여 초기 화면 간결화

### 5-6. 기술 정보 (현행 없음 → 신규)

현재 기술 컬럼들(비트레이트, 샘플레이트, 파일크기, ISRC, 발매일, 설명)은 PC에서 테이블 우측에, 모바일에서는 트랙별 아코디언에 표시.

## 6. 구현 순서

### Phase 1 — Hero Banner 교체
1. `_CSS`에서 `.album-header` 관련 스타일 제거
2. `.hero`, `.hero-bg`, `.hero-inner`, `.hero-cover`, `.hero-info` 등 신규 클래스 추가
3. `build_html()` 내 앨범 헤더 HTML 블록 교체
4. 커버아트 없을 때 폴백: 단색 그라디언트 배경 + 💿 이모지

### Phase 2 — 트랙 테이블 시각 개선 (PC)
1. 헤더 스타일 변경 (투명 배경, 하단 구분선)
2. 제목 셀을 2행 구조로 변경 (title + artist 서브텍스트)
3. 트랙번호 hover 효과 (CSS `.num` hover pseudo)
4. 타이틀 뱃지 / 가사 뱃지 스타일 교체

### Phase 3 — 모바일 카드 리스트
1. `.track-card-list` 추가 (모바일 전용, PC에서 `display: none`)
2. `.track-table-wrap` 모바일에서 `display: none`
3. `build_html()`에서 카드 리스트 HTML 블록 별도 생성
4. 트랙별 기술 정보 `<details>` 아코디언 생성

### Phase 4 — 앨범 소개 모바일 접기/펼치기
1. 모바일에서 `desc-box` → `<details class="desc-details">` 래핑
2. CSS `@media` 로 PC/모바일 표시 전환

### Phase 5 — 컬러 스킴 및 마무리
1. CSS 변수 값 업데이트
2. 다크모드 Hero 배경 최적화
3. 인쇄 스타일 업데이트 (Hero 배경 제거, 카드→테이블 전환)
4. eztag 배지 위치 및 푸터 최종 정리

## 7. 기술 제약 및 고려사항

### 자체 완결형 HTML 유지
- 외부 CDN 의존 없음 (폰트·아이콘 모두 인라인 또는 CSS)
- 커버아트 base64 인라인 유지

### Hero 배경 이미지
- 커버아트가 있는 경우: `background-image: url(커버 base64)` → CSS `filter: blur(40px)`
- 커버아트 없는 경우: 앨범 제목 첫 글자 기반 그라디언트 (JS로 색상 결정)
  ```javascript
  const colors = ['#667eea,#764ba2','#f093fb,#f5576c','#4facfe,#00f2fe',...];
  const idx = albumTitle.charCodeAt(0) % colors.length;
  ```

### 모바일 카드 vs 테이블 이중 렌더링
- 동일 데이터를 두 번 렌더링하므로 HTML 파일 크기 증가
- 트랙 수가 많을 경우 (100곡+) 허용 범위 내 (예상 추가 5~10KB)
- 기술 정보 컬럼(ISRC, 발매일 등)은 카드의 `<details>` 안에만 포함

### 다크모드 자동 감지
- 현행 `@media (prefers-color-scheme: dark)` 방식 유지
- Hero 배경은 다크모드에서 오버레이 불투명도를 더 높임

### 인쇄 최적화
- `@media print` 에서 Hero 배경 이미지 제거 (`display: none`)
- 모바일 카드 리스트 숨기고 테이블 표시로 강제 전환
- 나머지 현행 print 스타일 유지
