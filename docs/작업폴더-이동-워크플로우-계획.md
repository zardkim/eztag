# 작업폴더 → 대상폴더 이동 워크플로우 계획

> 작성일: 2026-03-18 · 대상 버전: eztag v0.5.x

---

## 1. 기능 개요

### 현재 방식의 문제점

| 항목 | 현재 방식 | 문제 |
|------|-----------|------|
| 스캔 범위 | NAS 전체 music 폴더 등록 | 수천 개 파일 로딩, 성능 저하 |
| 작업 흐름 | 완성된 파일과 미완성 파일 혼재 | 어디까지 작업했는지 파악 어려움 |
| 이동 | 수동 파일 탐색기 이동 | 프로그램 밖에서 별도 처리 필요 |

### 새로운 워크플로우

```
[NAS/외부 원본]
      │
      │ 복사/이동 (수동)
      ▼
[작업 폴더 data/library]  ← eztag 스캔 대상 (소량)
      │
      │ 태그 편집, 파일명 변경, 커버아트, LRC 등 작업
      ▼
[완료 버튼 클릭]
      │
      │ 대상 폴더 선택 (미리 등록된 경로)
      ▼
[대상 폴더 e.g. /volume1/music/팝] ← 실제 파일 이동
```

### 핵심 특징

| 항목 | 내용 |
|------|------|
| 작업 폴더 | `data/library` 하위 폴더 단위 작업 |
| 이동 대상 | 설정에서 미리 등록한 외부 경로 목록 |
| 이동 단위 | 폴더 단위 (선택한 폴더 전체) |
| 대상 브라우저 | 등록된 대상 경로의 하위 폴더 탐색 가능 |
| DB 처리 | 이동 후 DB 경로 자동 업데이트 |
| 캐시 처리 | 작업 폴더 캐시 무효화 + 스캔 폴더 갱신 |

## 2. 시스템 아키텍처

```
Frontend
  ├─ Settings.vue
  │    └─ 대상 폴더 등록/삭제 UI (DestinationFolders 섹션)
  │
  ├─ Browser.vue (FolderTree + 파일 목록)
  │    └─ 폴더 선택 시 "📦 이동" 버튼 표시
  │         └─ MoveToDestinationModal.vue 열기
  │
  └─ MoveToDestinationModal.vue
       ├─ 등록된 대상 폴더 목록
       ├─ 하위 폴더 탐색 (FolderPicker)
       ├─ 새 폴더 생성 옵션
       └─ 이동 실행 → POST /api/browse/move-folder

Backend
  ├─ Settings API: 대상 폴더 CRUD
  │    GET/POST/DELETE /api/config/destinations
  │
  └─ Browse API
       ├─ GET /api/browse/dest-children?path=... (대상 폴더 탐색)
       └─ POST /api/browse/move-folder
            ├─ 경로 검증 (작업 폴더 하위인지)
            ├─ 대상 폴더 검증 (등록된 경로 하위인지)
            ├─ shutil.move() 실행
            ├─ DB 경로 업데이트
            └─ 캐시 무효화
```

## 3. 설정: 대상 폴더 관리

### 3-1. 설정 화면 UI (Settings.vue → 일반 탭)

```
┌─────────────────────────────────────────────────────┐
│  📁 이동 대상 폴더                                   │
├─────────────────────────────────────────────────────┤
│  등록된 대상 폴더                                    │
│  ┌───────────────────────────────────────┬────────┐  │
│  │ /volume1/music                        │ [삭제] │  │
│  │ /volume1/music/클래식                 │ [삭제] │  │
│  │ /volume2/backup/music                 │ [삭제] │  │
│  └───────────────────────────────────────┴────────┘  │
│                                                     │
│  새 대상 폴더 추가                                   │
│  [ /volume1/music/팝          ] [+ 추가]            │
│                                                     │
│  ※ 대상 폴더는 Docker 볼륨으로 마운트된 경로여야    │
│     합니다. (예: /dest/music → NAS /volume1/music)  │
└─────────────────────────────────────────────────────┘
```

### 3-2. Docker 볼륨 마운트 방식

NAS의 `/volume1/music`을 컨테이너에서 `/dest/music`으로 마운트:

```yaml
# docker-compose.yml
volumes:
  - /volume1/music:/dest/music
  - /volume2/backup:/dest/backup
```

설정에서 `/dest/music` 등록 → 실제로는 NAS `/volume1/music`에 저장됨.

## 4. 이동 모달: MoveToDestinationModal.vue

```
┌─────────────────────────────────────────────────────────┐
│  📦 폴더 이동: "Ed Sheeran - Divide"             [×]   │
├─────────────────────────────────────────────────────────┤
│  이동할 폴더                                            │
│  📁 Ed Sheeran - Divide  (12개 파일, 245 MB)           │
├─────────────────────────────────────────────────────────┤
│  대상 위치 선택                                         │
│                                                         │
│  ▼ /dest/music  (NAS: /volume1/music)                   │
│     ├─ 📁 팝                         [선택]             │
│     │   ├─ 📁 국내팝                 [선택]             │
│     │   └─ 📁 팝팝            [선택]             │
│     ├─ 📁 클래식                     [선택]             │
│     └─ 📁 재즈                       [선택]             │
│                                                         │
│  ▼ /dest/backup  (NAS: /volume2/backup)                 │
│     └─ 📁 music                      [선택]             │
│                                                         │
│  ➕ 새 폴더 만들기: [ 팝/영국팝      ] [생성]           │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  이동 대상: /dest/music/팝                              │
│  (NAS 실경로: /volume1/music/팝/Ed Sheeran - Divide)   │
│                                                         │
│  [취소]                          [📦 여기로 이동]       │
└─────────────────────────────────────────────────────────┘
```

### 4-1. 폴더 탐색 동작

- 등록된 대상 폴더 목록을 루트로 표시
- 각 폴더 클릭 시 하위 폴더 lazy load (`GET /api/browse/dest-children?path=...`)
- **[선택]** 버튼으로 이동 대상 확정
- **새 폴더 만들기**: 경로 입력 후 생성 → 대상으로 자동 선택

### 4-2. 이동 확인

- 선택된 대상 경로 표시
- NAS 실경로 (레이블) 병기
- 이동 후 결과: 동일 폴더명 존재 시 경고 표시

## 5. 백엔드 구현

### 5-1. 대상 폴더 설정 API (`config.py`)

```python
# 대상 폴더 목록 조회
GET /api/config/destinations
Response: [
  { "path": "/dest/music", "label": "NAS 음악 폴더" },
  ...
]

# 대상 폴더 추가
POST /api/config/destinations
Body: { "path": "/dest/music", "label": "NAS 음악 폴더" }

# 대상 폴더 삭제
DELETE /api/config/destinations/{index}
```

설정 저장: `AppConfig` 테이블의 `destination_folders` 키 (JSON 배열)

### 5-2. 대상 폴더 탐색 API (`browse.py`)

```python
# 대상 폴더 하위 목록 (등록된 경로 하위만 허용)
GET /api/browse/dest-children?path=/dest/music
Response: [
  { "name": "팝", "path": "/dest/music/팝", "has_children": true },
  { "name": "클래식", "path": "/dest/music/클래식", "has_children": false },
]
```

### 5-3. 폴더 이동 API (`browse.py`)

```python
POST /api/browse/move-folder
Body: {
  "source_path": "/app/data/library/Ed Sheeran - Divide",
  "dest_path": "/dest/music/팝"
}
Response: {
  "ok": true,
  "new_path": "/dest/music/팝/Ed Sheeran - Divide",
  "moved_files": 12,
  "error": null
}
```

### 5-4. 이동 처리 로직

```
1. 검증
   a. source_path가 작업 폴더(data/library) 하위인지 확인
   b. dest_path가 등록된 대상 폴더 하위인지 확인
   c. 대상에 동일 폴더명 존재 여부 확인

2. 이동 실행
   a. shutil.move(source_path, dest_path) → 폴더째 이동
   b. 이동 성공 시 DB 처리:
      - Track.file_path 일괄 업데이트 (source_path → new_path)
      - 작업 폴더 스캔 폴더에서 경로 정리 (선택적)

3. 캐시 무효화
   a. 작업 폴더 캐시 삭제
   b. 브라우저 파일 목록 갱신

4. 결과 반환
```

### 5-5. 새 폴더 생성 API

```python
POST /api/browse/dest-mkdir
Body: { "parent_path": "/dest/music/팝", "name": "영국팝" }
Response: { "ok": true, "path": "/dest/music/팝/영국팝" }
```

## 9. 구현 순서 (단계별)

### Phase 1 — 설정: 대상 폴더 관리

3. Settings.vue → 일반 탭에 대상 폴더 관리 섹션 추가

5. `POST /api/browse/dest-mkdir` — 새 폴더 생성
6. `POST /api/browse/move-folder` — 폴더 이동 + DB 업데이트

### Phase 3 — 프론트엔드: 이동 모달
7. `MoveToDestinationModal.vue` 컴포넌트
   - 대상 폴더 트리 탐색
   - 새 폴더 생성
   - 이동 실행
8. `Browser.vue` 툴바에 `📦 이동` 버튼 추가 (작업 폴더 하위일 때만)
9. `api/index.js`에 API 메서드 추가

### Phase 4 — 연동 완성
10. 이동 후 파일 목록/폴더 트리 자동 갱신
11. 성공/실패 토스트 + 에러 처리
12. i18n 키 완성 (ko/en)
