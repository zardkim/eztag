# Workspace + Library 이중 폴더 워크플로우 개선 계획

## 1. 배경 및 목표

### 실제 음악 정리 워크플로우

사용자의 음악 파일 정리 패턴은 크게 두 가지다.

**패턴 A: 작업 후 이동**
```
다운로드/리핑 → [Workspace] 태그 편집 → [Library] 폴더 이동
```

**패턴 B: 이동 후 작업**
```
다운로드/리핑 → [Library] 폴더 이동 → [Library] 태그 편집
```

두 패턴 모두 지원하려면:
- **Workspace**: 정리되지 않은 파일을 편집하는 작업 공간
- **Library**: 정리 완료된 음악 보관 폴더
- 두 영역 간 **폴더 이동** 기능
- 두 영역 모두에서 **태그 편집** 가능

---

## 2. 현재 상태 분석

### Docker 마운트 (현재)
```
/music  ←  ${MUSIC_PATH}  (read-only, 라이브러리)
```
- 라이브러리만 마운트되어 있음
- read-only라서 태그 쓰기 불가 (실제로는 rw로 마운트하는 경우도 있음)
- 워크스페이스 개념 없음

### 백엔드 경로 변수 (현재)
| 환경변수 | 기본값 | 용도 |
|---|---|---|
| `MUSIC_BASE_PATH` | `/music` | 라이브러리 루트 |
| `COVERS_PATH` | `/app/data/covers` | 커버 이미지 저장 |

### 프론트엔드 (현재)
- `Browser.vue` — 임의 폴더를 열어 태그 편집
- 사이드바에서 폴더 선택 후 작업
- `MoveToDestinationModal` — 폴더를 설정된 대상으로 이동
- 라이브러리/워크스페이스 구분 없음

---

## 3. 목표 아키텍처

### Docker 마운트 (목표)
```
/music      ←  ${MUSIC_PATH}      (라이브러리, rw)
/workspace  ←  ${WORKSPACE_PATH}  (워크스페이스, rw)
```

### 백엔드 환경변수 추가
| 환경변수 | 기본값 | 용도 |
|---|---|---|
| `MUSIC_BASE_PATH` | `/music` | 라이브러리 루트 (유지) |
| `WORKSPACE_PATH` | `/workspace` | 워크스페이스 루트 (신규) |

### 폴더 역할 정의
| 영역 | 경로 | 역할 | 읽기 | 쓰기 | 태그편집 | 폴더이동 |
|---|---|---|---|---|---|---|
| Workspace | `/workspace` | 다운로드/리핑 원본 | ✅ | ✅ | ✅ | → Library |
| Library | `/music` | 정리된 음악 보관 | ✅ | ✅ | ✅ | 내부 이동 |

## 4. 필요 기능 상세

### 4-1. 사이드바 개편

**현재**: 임의 폴더 열기 버튼 하나
**목표**: Workspace / Library 두 섹션으로 구분

```
┌─────────────────────────┐
│ 🏠 홈                    │
├─────────────────────────┤
│ WORKSPACE               │
│ [📂 워크스페이스 열기]    │
│ ├ Album A               │
│ ├ Album B               │
│ └ Album C               │
├─────────────────────────┤
│ LIBRARY                 │
│ [📚 라이브러리 열기]      │
│ ├ Artist 1/             │
│ └ Artist 2/             │
└─────────────────────────┘
```

- 앱 시작 시 워크스페이스 루트를 자동으로 열기 (설정 가능)
- 각 섹션에서 하위 폴더 탐색 가능
- 현재 선택된 폴더가 어느 영역인지 구분 표시

### 4-2. 워크스페이스 → 라이브러리 폴더 이동

**현재**: `MoveToDestinationModal`에서 `destination_folders` 설정 기반 이동
**목표**: 워크스페이스 폴더를 라이브러리로 이동하는 전용 워크플로우

동작 흐름:
1. 워크스페이스에서 폴더 선택 (태그 편집 완료 상태)
2. "라이브러리로 이동" 버튼 클릭
3. 라이브러리 폴더 트리에서 이동 위치 선택
4. 이동 확인 → 서버에서 폴더 이동 실행
5. 이동 후 라이브러리 폴더로 자동 이동 (선택)

이동 대상 선택 UI:
```
라이브러리로 이동
├ /music/
│ ├ K-POP/
│ │ ├ IU/          ← 선택
│ │ └ BTS/
│ └ Jazz/
└ [새 폴더 만들기]
```

### 4-3. 라이브러리 내 폴더 이동

라이브러리 내에서도 폴더 위치를 변경할 수 있어야 함 (현재 `MoveToDestinationModal`로 부분 지원).
`destination_folders` 설정에 의존하지 않고 라이브러리 트리를 직접 탐색해 이동 위치 지정.

### 4-4. 기본 시작 폴더 설정

설정에서 앱 시작 시 자동으로 열 폴더 지정:
- `startup_folder`: `workspace` (기본) | `library` | `none`

### 4-5. 폴더 영역 구분 표시

현재 열린 폴더가 워크스페이스인지 라이브러리인지 명확히 표시:
- 경로 바 색상 구분 (예: 워크스페이스=주황, 라이브러리=파란)
- 툴바에 현재 영역 뱃지 표시

## 5. 구현 계획

### Phase 1 — Docker & 백엔드 환경 설정

**docker-compose.yml 변경**
```yaml
backend:
  environment:
    MUSIC_BASE_PATH: /music
    WORKSPACE_PATH: /workspace      # 신규
  volumes:
    - ${MUSIC_PATH:-./data/library}:/music    # rw로 변경
    - ${WORKSPACE_PATH:-./data/workspace}:/workspace  # 신규
```

**backend/app/config.py 변경**
```python
WORKSPACE_PATH: str = "/workspace"   # 신규
```

**backend/app/core/config_store.py 변경**
```python
"startup_folder": ("workspace", "앱 시작 시 자동으로 열 폴더 (workspace/library/none)")
```

**backend/app/api/browse.py 변경**
- `_validate_path()` 에서 허용 경로에 `WORKSPACE_PATH` 추가
- `/workspace/roots` 엔드포인트 추가 (워크스페이스 루트 목록)
- `/workspace/children` 엔드포인트 추가 (워크스페이스 하위 폴더)
- `/move-to-library` 엔드포인트 추가 (워크스페이스 → 라이브러리 이동)

**backend/app/api/workspace.py (기존 활용)**
- 현재 `library/roots`, `library/children`만 제공
- `workspace/roots`, `workspace/children` 추가

### Phase 2 — 프론트엔드 사이드바 개편

**WorkspaceSidebar.vue 개편**
- Workspace 섹션: 워크스페이스 루트 폴더 목록 및 탐색
- Library 섹션: 라이브러리 루트 폴더 목록 및 탐색
- 각 섹션 독립적으로 열기/닫기

**browserStore 개선**
- `currentArea`: `'workspace' | 'library' | null` — 현재 열린 폴더의 영역
- `workspaceRoot`: 워크스페이스 루트 경로
- `libraryRoot`: 라이브러리 루트 경로

**API 추가**
```js
export const workspaceApi = {
  libraryRoots: () => client.get('/workspace/library/roots'),
  libraryChildren: (path) => client.get('/workspace/library/children', { params: { path } }),
  workspaceRoots: () => client.get('/workspace/workspace/roots'),    // 신규
  workspaceChildren: (path) => client.get('/workspace/workspace/children', { params: { path } }),  // 신규
}
```

### Phase 3 — 폴더 이동 워크플로우

**MoveToLibraryModal.vue (신규)**
- 워크스페이스 폴더를 라이브러리로 이동하는 전용 모달
- 라이브러리 폴더 트리 탐색 (현재 `LibraryPickerModal` 참고)
- 새 폴더 만들기 지원
- 이동 후 라이브러리 해당 폴더로 자동 이동 옵션

**기존 MoveToDestinationModal 유지**
- 라이브러리 내부 이동 용도로 계속 사용 가능
- 또는 동일한 MoveToLibraryModal로 통합

```python
    dest_path: str,        # 라이브러리 내 대상 경로
):
    # 경로 검증 (source는 workspace 내, dest는 library 내)
    # shutil.move 실행
    # 성공 시 새 경로 반환
```

### Phase 4 — 영역 구분 UI

**Browser.vue 경로 바 개선**
```html
<!-- 워크스페이스 폴더인 경우 -->
<span class="badge bg-orange-100 text-orange-600">Workspace</span>

<!-- 라이브러리 폴더인 경우 -->
<span class="badge bg-blue-100 text-blue-600">Library</span>
```

**툴바 버튼 조건 표시**
- "라이브러리로 이동" 버튼: 워크스페이스 폴더일 때만 표시
- "폴더 이동" 버튼: 라이브러리 폴더일 때 표시 (내부 이동)

### Phase 5 — 설정 및 초기화

**Settings.vue 추가**
- 시작 폴더 설정: `워크스페이스` / `라이브러리` / `없음`
- 워크스페이스 경로 표시 (읽기 전용 — 환경변수에서)
- 라이브러리 경로 표시 (읽기 전용 — 환경변수에서)

**앱 시작 시 자동 폴더 열기 (App.vue)**
```js
onMounted(async () => {
  const startupFolder = await configApi.get('startup_folder')
  if (startupFolder === 'workspace') {
    browserStore.openWorkspaceRoot()
  } else if (startupFolder === 'library') {
    browserStore.openLibraryRoot()
  }
})
```

```yaml
      - eztag-net

      - backend
```

## 7. 구현 우선순위

| 우선순위 | 항목 | 난이도 | 설명 |
|---|---|---|---|
| 1 | Docker + 백엔드 경로 추가 | 낮음 | `WORKSPACE_PATH` 환경변수, 경로 검증 |

| 6 | 라이브러리 내 폴더 이동 개선 | 중간 | 기존 MoveToDestinationModal 확장 |

## 8. 고려 사항

- **경로 보안**: `_validate_path()`에서 workspace/library 범위 밖 접근 차단 유지
- **라이브러리 스캔**: 워크스페이스는 스캔 대상에서 제외 (라이브러리만 DB에 인덱싱)
- **이동 후 처리**: 라이브러리로 이동된 폴더는 다음 스캔 시 자동으로 DB에 추가됨
- **워크스페이스 없는 경우**: `WORKSPACE_PATH`가 설정되지 않으면 워크스페이스 섹션 숨김, 기존 단일 폴더 모드로 동작
- **커버 이미지**: 워크스페이스 파일의 커버도 동일하게 추출/표시 가능 (이미 지원)
