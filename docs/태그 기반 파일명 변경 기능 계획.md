# 태그 기반 파일명 변경 기능 계획

> 참조: `docs/태그 필드 매핑.md` · Mp3tag 호환 포맷 스트링
> 작성일: 2026-03-11 · 대상 버전: eztag v0.4.x

---

## 1. 기능 개요

Browser 뷰에서 선택한 파일(단일 또는 복수)의 태그 정보를 기반으로
사용자가 입력한 **포맷 패턴**에 따라 파일명을 자동 생성·변경하는 기능.

### 핵심 특징

| 항목 | 내용 |
|------|------|
| 포맷 스트링 | Mp3tag 호환 `%field%` 변수 방식 |
| 적용 단위 | 체크박스 선택 파일 (복수) / 현재 폴더 전체 |
| 미리보기 | 패턴 입력 즉시 변경 전/후 파일명 테이블 표시 |
| 안전 처리 | 특수문자 자동 치환, 중복 파일명 감지 |
| 제로패딩 | 트랙 번호 등 숫자형 필드 자릿수 지정 |
| 캐시 무효화 | 변경 성공 시 해당 폴더 캐시 자동 갱신 |

---

## 2. 지원 포맷 변수

`docs/태그 필드 매핑.md` 3장 기준, 파일명 변경에 활용 가능한 변수 목록:

### 태그 기반 변수

| 포맷 변수 | eztag 필드 | 예시 |
|-----------|-----------|------|
| `%title%` | `title` | `Shape of You` |
| `%artist%` | `artist` | `Ed Sheeran` |
| `%albumartist%` | `album_artist` | `Ed Sheeran` |
| `%album%` | `album_title` | `Divide` |
| `%track%` | `track_no` | `3` → `03` (2자리 패딩) |
| `%totaltracks%` | `total_tracks` | `16` |
| `%disc%` | `disc_no` | `1` |
| `%year%` | `year` | `2017` |
| `%genre%` | `genre` | `Pop` |
| `%publisher%` | `label` | `Atlantic Records` |

### 파일 속성 변수

| 포맷 변수 | 내용 | 예시 |
|-----------|------|------|
| `%_filename%` | 현재 파일명 (확장자 제외) | `01 Shape of You` |
| `%_ext%` | 확장자 | `mp3` |
| `%_bitrate%` | 비트레이트 (kbps) | `320` |
| `%_codec%` | 코덱 | `MP3` |

### 패딩 수식자

- `%track%` → 기본 2자리 제로패딩 (`1` → `01`, `12` → `12`)
- `%disc%` → 기본 패딩 없음
- 미래 확장: `%track:2%`, `%disc:1%` 형식으로 자릿수 지정 가능

## 3. 포맷 패턴 예시

```
# 트랙번호-제목
%track% - %title%
→ 03 - Shape of You.mp3

# 아티스트-트랙번호-제목
%artist% - %track% - %title%
→ Ed Sheeran - 03 - Shape of You.mp3

# 앨범아티스트/연도 - 앨범/트랙번호 - 제목 (폴더 구분 없이 평탄화)
%albumartist% - %year% - %album% - %track% - %title%
→ Ed Sheeran - 2017 - Divide - 03 - Shape of You.mp3

# 디스크-트랙번호-제목
%disc%-%track% - %title%
→ 1-03 - Shape of You.mp3
```

### 사전 정의 프리셋

| 프리셋 이름 | 패턴 |
|------------|------|
| 트랙 - 제목 | `%track% - %title%` |
| 아티스트 - 제목 | `%artist% - %title%` |
| 트랙 - 아티스트 - 제목 | `%track% - %artist% - %title%` |
| 아티스트 - 앨범 - 트랙 - 제목 | `%artist% - %album% - %track% - %title%` |
| 디스크-트랙 - 제목 | `%disc%-%track% - %title%` |


## 5. 백엔드 구현

### 5-1. 유틸리티 모듈: `backend/app/core/pattern_rename.py`

```python
"""태그 기반 파일명 패턴 처리 유틸리티."""
import re

# 파일명에 사용 불가한 문자 → 치환 대상
INVALID_CHARS = r'[\\/:*?"<>|]'

def render_pattern(pattern: str, fields: dict) -> str:
    """
    %field% 변수를 실제 태그 값으로 치환.

    - track_no: 2자리 제로패딩
    - None 값: 빈 문자열로 대체
    - 결과 양 끝 공백/점/구분자 제거
    """
    ...

def sanitize_filename(name: str) -> str:
    """
    파일명 안전 처리:
    - 금지 문자(/ \\ : * ? " < > |) → '_' 치환
    - 연속 공백 → 단일 공백
    - 앞뒤 공백/점 제거
    - 빈 문자열 → '_' 대체
    """
    ...

def build_new_name(pattern: str, file_info: dict, extension: str) -> str:
    """
    패턴 + 파일 정보 → 최종 새 파일명 (확장자 포함).
    """
    name = render_pattern(pattern, file_info)
    name = sanitize_filename(name)
    return f"{name}.{extension.lstrip('.')}"
```

### 5-2. API 엔드포인트: `backend/app/api/browse.py`

```python
# 미리보기 (파일 시스템 변경 없음)
POST /api/browse/rename-by-tags/preview
Request:
  {
    "paths": ["/music/01 old.mp3", ...],
    "pattern": "%track% - %title%"
  }
Response:
  [
    {
      "path": "/music/01 old.mp3",
      "old_name": "01 old.mp3",
      "new_name": "01 - Shape of You.mp3",
      "conflict": false,   # 동일 폴더에 이미 존재하는 이름인지
      "error": null        # 오류 메시지 (태그 없음 등)
    },
    ...
  ]

# 실제 변경
POST /api/browse/rename-by-tags
Request:
  {
    "paths": ["/music/01 old.mp3", ...],
    "pattern": "%track% - %title%"
  }
Response:
  {
    "ok": true,
    "results": [
      { "path": "/music/01 old.mp3", "new_path": "/music/01 - Shape of You.mp3", "ok": true },
      { "path": "/music/02 old.mp3", "new_path": null, "ok": false, "error": "conflict" },
      ...
    ],
    "success": 8,
    "failed": 1
  }
```

### 5-3. 처리 로직 (rename-by-tags)

```
1. 각 path에 대해:
   a. _validate_path() — 스캔 폴더 하위인지 검증
   b. read_tags() 또는 DB 조회로 태그 필드 수집
   c. render_pattern() + sanitize_filename() → 새 파일명 생성
   d. 새 경로 = 같은 폴더 + 새 파일명 + 원본 확장자
   e. 충돌 감지: new_path.exists() and new_path != old_path
   f. conflict=false 인 경우에만 os.rename() 실행
   g. DB Track.file_path 업데이트
   h. _cache.invalidate_for_file() 호출

2. 성공/실패 결과 반환
```

### 5-4. 태그 소스 우선순위

```
1. DB에 있고 최신(mtime 기준)이면 DB 값 사용
2. DB 없거나 오래됐으면 read_tags() (파일 직접 읽기)
```

## 6. 프론트엔드 구현

### 6-1. 진입점 (Browser.vue 툴바)

- 체크박스로 파일 선택 후 **"파일명 변경"** 버튼 클릭
- 미선택 시 현재 폴더 전체 파일 적용
- 버튼은 `browserStore.checkedFiles.length > 0` 또는 `files.length > 0` 일 때 표시

### 6-2. 모달 컴포넌트: `RenameByTagsModal.vue`

```
┌─────────────────────────────────────────────────────┐
│  태그 기반 파일명 변경                           [×] │
├─────────────────────────────────────────────────────┤
│  패턴: [%track% - %title%________________] [▾ 프리셋]│
│                                                     │
│  변수: %title% %artist% %album% %track% %year% ...  │
│        (클릭하면 패턴 입력창에 삽입)                │
├─────────────────────────────────────────────────────┤
│  미리보기 (8개 파일)                                │
│  ┌────────────────────┬────────────────────┬──────┐ │
│  │ 현재 파일명        │ 변경될 파일명       │ 상태 │ │
│  ├────────────────────┼────────────────────┼──────┤ │
│  │ 01 old.mp3         │ 01 - Title.mp3      │  ✅  │ │
│  │ 02 old.mp3         │ 02 - Title2.mp3     │  ✅  │ │
│  │ 03 old.mp3         │ 03 - Title3.mp3     │ ⚠️충돌│ │
│  │ 04 no_tag.mp3      │ (태그 없음)         │  ❌  │ │
│  └────────────────────┴────────────────────┴──────┘ │
│                                                     │
│  ✅ 6개 변경 가능  ⚠️ 1개 충돌  ❌ 1개 태그 없음    │
│                                                     │
│         [취소]          [적용 (6개 파일)]           │
└─────────────────────────────────────────────────────┘
```

### 6-3. 상태 관리

- 패턴 입력 시 **디바운스 300ms** 후 미리보기 자동 갱신
- 미리보기는 **프론트엔드 로컬**에서 계산 (현재 파일 목록의 태그 값 활용)
  - 단, 최초 모달 오픈 시 `/preview` API 호출하여 서버측 검증 병행
- 적용 버튼: conflict=false 인 파일만 대상으로 확정

### 6-4. API 클라이언트 (`api/index.js`)

```javascript
export const browseApi = {
  // 기존
  rename: (data) => client.post('/browse/rename', data),

  // 신규
  renameByTagsPreview: (data) => client.post('/browse/rename-by-tags/preview', data),
  renameByTags: (data) => client.post('/browse/rename-by-tags', data),
}
```

## 7. 파일명 안전 처리 규칙

| 상황 | 처리 방법 |
|------|----------|
| `/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `\|` | `_` 로 치환 |
| 태그 값이 `None` / 비어있음 | 해당 변수를 빈 문자열로 대체 |
| 패턴 결과가 빈 문자열 | `_` 로 대체 |
| 앞뒤 공백, 마침표 | strip |
| 연속 공백 | 단일 공백으로 압축 |
| 결과 파일명이 원본과 동일 | 변경 없이 건너뜀 (성공 처리) |
| 동일 폴더에 이미 같은 이름 존재 | conflict 표시, 해당 파일 건너뜀 |
| 길이 255바이트 초과 | 초과 부분 잘라내기 |
